## Что было сделано? 

### ✅ Backend (C# / ASP.NET Core)

**Добавлены 4 новые модели запчастей:**
- **Motor** - Моторы (мощность, тип, цена)

- TiresController.cs
- BodiesController.cs

**Каждый контроллер имеет полный CRUD:**
- GET (получить все / по ID)

- PUT (обновить)
- DELETE (удалить)

## 🧪 Тестирование

### Тест через браузер
1. Откройте http://localhost:5173
2. Нажмите на "Моторы"
3. Введите данные мотора
4. Нажмите "Добавить мотор"
5. Мотор появится в списке
6. Нажмите "Редактировать" для изменения
7. Нажмите "Удалить" для удаления

### Тест через API (PowerShell)

**Получить все моторы:**
```powershell
curl.exe http://localhost:5026/api/motors
```

**Добавить мотор:**
```powershell
curl.exe -X POST http://localhost:5026/api/motors `
  -H "Content-Type: application/json" `
  -d '{"name":"V8","power":400,"type":"Бензин","price":5000}'
```

## 💡 Интересные факты

1. **Generic сервис** - Один базовый класс `PartsService<T>` используется всеми 4 сервисами
2. **Thread-safe** - Все операции с файлами защищены блокировками
3. **Атомарные операции** - Используются временные файлы для безопасной записи
4. **CORS включена** - Frontend и backend могут общаться свободно
5. **Нет базы данных** - Всё хранится в JSON для простоты

## ❓ Если что-то не работает

### API не отвечает?
- Убедитесь, что backend запущен: http://localhost:5026/api/motors
- Проверьте, что порт 5026 свободен

### Frontend не загружается?
- Убедитесь, что npm install выполнен
- Проверьте, что npm run dev запущен
- Очистите кэш браузера (Ctrl+Shift+Del)

### Данные не сохраняются?
- Проверьте, что папка data/ создана в bin/Debug/net10.0/
- Убедитесь, что у файлов есть права на запись

### JSON файлы не созданы?
- Они создаются автоматически при первом запуске backend
- Если их нет, проверьте права на папку data/
