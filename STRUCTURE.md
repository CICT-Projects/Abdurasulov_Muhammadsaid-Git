## 📋 Описание ключевых файлов

### Бэкенд

#### `TaskApi/Program.cs` (100+ строк)
Содержит:
- **Модель `Car`**: Класс с полями Id, Brand, Model, Year, Color
- **Сервис `CarService`**: Логика CRUD операций для машин (хранит в памяти)
- **Конфигурация приложения**:
  - OpenAPI (Swagger)
  - CORS для React приложения
  - Регистрация сервисов
  - Middleware
  - Маршрутизация контроллеров

#### `TaskApi/Controllers/CarsController.cs` (~80 строк)
REST контроллер с endpoints:
- `GET /api/cars` - получить все машины
- `GET /api/cars/{id}` - получить машину по ID
- `POST /api/cars` - создать машину
- `PUT /api/cars/{id}` - обновить машину
- `DELETE /api/cars/{id}` - удалить машину

### Фронтенд

#### `frontend/src/App.jsx` (~200 строк)
Главный React компонент:
- State для хранения машин и данных формы
- useEffect для загрузки данных при монтировании
- Функции для CRUD операций (fetch запросы)
- Форма для добавления/редактирования машин
- Отображение списка машин в виде галереи

#### `frontend/src/App.css` (~200 строк)
Стили для:
- Градиентного фона (фиолетовый)
- Формы ввода с фокусом
- Карточек машин с анимациями
- Кнопок действий (добавить, редактировать, удалить)
- Адаптивного макета для мобильных устройств

#### `frontend/src/index.css` (~20 строк)
Глобальные стили для всего приложения

### Конфигурация

#### `HomeTask.sln`
Visual Studio Solution файл для открытия обоих проектов одновременно

#### `run.bat`
Батник для запуска обоих приложений одновременно на Windows

#### `appsettings.json` и `appsettings.Development.json`
Конфигурация .NET приложения (логирование, базы данных и т.д.)

#### `package.json`
NPM зависимости и скрипты для React приложения

## 📝 Как добавить новый файл?

### Новый контроллер в бэкенде:
1. Создайте `TaskApi/Controllers/NewController.cs`
2. Добавьте `[ApiController]` и `[Route("api/[controller]")]`
3. Наследуйтесь от `ControllerBase`
4. Добавьте методы с `[HttpGet]`, `[HttpPost]` и т.д.

### Новый компонент на фронтенде:
1. Создайте `frontend/src/components/NewComponent.jsx`
2. Экспортируйте как default export
3. Импортируйте в `App.jsx` и используйте
